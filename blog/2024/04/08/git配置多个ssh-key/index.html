    Git配置多个SSH Key

    什么是SSH-Key？Git如何配置多个SSH-Key？

    SSH-Key是什么？

    Git的SSH密钥用于在客户端和远程仓库之间建立安全的通信连接。具体来说，SSH（Secure Shell）密钥对是一种公钥加密方法，它包括一个私钥和一个公钥。

    私钥（Private Key）

    私钥是保密的，它存储在你的本地计算机上。私钥用于在与远程服务器通信时对数据进行加密和签名，以证明你的身份。私钥应该始终保持私密，不泄露给任何人。

    公钥（Public Key）

    公钥是公开的，可以安全地分享给任何需要验证你身份的服务器或服务。公钥用于解密由你的私钥加密的数据，或者验证由私钥签名的数据确实是由你的私钥创建的。

    Git的SSH密钥工作原理

    Git的SSH密钥基于公钥加密原理，使用一对密钥来实现安全的数据传输和身份验证：一个私钥和一个公钥。

    1. 密钥生成： 当你使用ssh-keygen命令生成SSH密钥对时，它会创建一个私钥和一个公钥。私钥存储在你的本地计算机上的~/.ssh目录中，而公钥可以被复制并放置在远程服务器上。

    2. 公钥分发： 公钥的目的是让远程服务器能够验证你的身份。在Git的情况下，你会将公钥添加到远程仓库服务（如GitHub、GitLab等）的账户设置中。这样，当你尝试与远程仓库通信时，服务器可以使用这个公钥来验证你的身份。

    3. 身份验证过程： 当你尝试通过SSH连接到远程服务器（例如，执行git push或git pull）时，服务器会发送一个随机生成的质询（challenge）给你的客户端。你的客户端使用私钥对这个质询进行加密，然后将加密后的响应发送回服务器。

    4. 验证响应： 服务器收到加密后的响应后，会使用你的公钥尝试解密这个响应。如果解密成功，并且解密后的质询与原始的质询相匹配，服务器就会确认你的身份，并允许你访问远程仓库。这个过程称为数字签名验证。

    为什么使用SSH密钥

    • 安全性：SSH密钥提供了比传统密码更强的安全性。即使有人截获了加密的数据，没有相应的私钥也无法解密和访问数据。
    • 方便性：一旦设置了SSH密钥，你就可以在不输入密码的情况下安全地与远程仓库通信。
    • 控制访问：你可以为不同的远程仓库设置不同的SSH密钥，从而实现细粒度的访问控制。

    通过使用SSH密钥，Git可以确保只有经过授权的用户才能访问和修改远程仓库中的数据，同时保证了通信的安全性和私密性。

    Git配置多个SSH-Key

    以github和gitee为例。

    1. 生成第一个 SSH-Key

    bash ssh-keygen -t rsa -C 'xx@163.com' -f ~/.ssh/gitee_id_rsa`

    1. 生成第二个 SSH-Key

    bash `ssh-keygen -t rsa -C 'xx@outlook.com' -f ~/.ssh/github_id_rsa`

    其中：

    • -t：key 类型
    • -C：注释

    • -f：指定生成的文件名

    • 在 ~/.ssh 目录下新建一个 config 文件

    # gitee
Host gitee.com
HostName gitee.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/gitee_id_rsa

# github
Host github.com
HostName github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/github_id_rsa

    ~/. 目录下应有如下文件：

    bash (base) gjd@MBP .ssh % cd ~/.ssh (base) gjd@MBP .ssh % ls config gitee_id_rsa.pub github_id_rsa.pub known_hosts.old gitee_id_rsa github_id_rsa known_hosts

    1. 在Gitee 和 Github中添加公钥

    1. 测试是否成功
    ssh -T git@gitee.com
ssh -T git@github.com 

# 成功提示：
# Hi ${username}! You've successfully authenticated, but GitHub does not provide shell access.

    配置多个SSH-Key后，实际使用过程中，Git会根据推送或拉取的远程仓库URL来选择正确的密钥，无需手动切换。
